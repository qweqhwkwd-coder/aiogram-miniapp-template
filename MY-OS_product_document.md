# MY-OS — Product Document v1.0
> Личная операционная система. Telegram Mini App + Bot.

---

## 1. Концепция

**MY-OS** — это не планировщик и не трекер. Это персональная ОС для жизни: всё что ты делаешь, ешь, тренируешь, думаешь и планируешь — в одном месте, с AI-слоем поверх.

**Ключевая идея:** каждое действие в приложении влияет на твои RPG-характеристики. Ты не просто ставишь галочки — ты прокачиваешь персонажа, которым являешься.

**Аудитория v1:** только ты. Архитектура с первого дня строится под мульти-пользователей — чтобы потом можно было продавать/делиться.

### 04 · ТРЕНИРОВКИ + AI-ТРЕНЕР

*(Нового модуля нет в MEOS — наш собственный)*

#### 4.1 Лог активности

**Концепция:** нет фиксированных типов из списка. Ты сам создаёшь свои активности — один раз, потом одним тапом.

---

**Три пути добавления тренировки:**

**1. Быстрый тап** (основной способ)

Твои сохранённые активности отображаются как кнопки:
```
[🧗 Боулдеринг] [🏊 Плавание] [💪 Зал]
[🥋 Джиу] [🏃 Бег] [🚴 Велосипед] [＋]
```
Нажал → AI уже знает структуру этой активности → задаёт только релевантные вопросы:
- Боулдеринг: "Сколько трасс? Какие уровни?"
- Плавание: "Дистанция? Стиль? Темп?"
- Зал: "Какие упражнения сегодня?"

---

**2. Свободный ввод** (для новой или разовой активности)

Пишешь или говоришь что угодно в свободной форме:
```
"час джиу, много партерной работы, устал"
"утренняя пробежка 8км"
"йога 40 минут, фокус на бёдра"
"футбол с друзьями"
```
AI анализирует → формирует структуру под конкретную активность → задаёт уточняющие вопросы только если нужно.

После сохранения: **"Сохранить как шаблон?"** → если да, активность появляется в быстрых кнопках.

---

**3. Создать шаблон вручную**

Нажал [＋] → вводишь название, выбираешь категорию (кардио / силовое / смешанное / гибкость), описываешь какие поля нужны.

---

**Категоризация для RPG и статистики:**

При первом вводе новой активности AI определяет категорию автоматически. Если неуверен — спрашивает один раз:
```
"Боулдеринг — это больше кардио, силовое или смешанное?"
```
Ответ запоминается навсегда для этой активности. Следующий раз — без вопроса.

| Категория | RPG характеристика |
|---|---|
| Кардио | Выносливость |
| Силовое | Сила |
| Смешанное | Выносливость + Сила (50/50) |
| Гибкость / подвижность | Рефлексия |

---

**Хранение в БД:**

```sql
-- Шаблоны активностей пользователя
activity_templates (
  id, user_id,
  name,           -- "Боулдеринг"
  emoji,          -- "🧗"
  category,       -- "mixed" | "cardio" | "strength" | "flexibility"
  fields_schema,  -- jsonb: какие поля собирать
  sort_order,     -- порядок кнопок
  use_count,      -- сколько раз использован
  created_at
)

-- Лог тренировок
workouts (
  id, user_id,
  template_id,       -- ссылка на шаблон (null если разовая)
  activity_name,     -- название (копия из шаблона или свободный ввод)
  category,
  date, duration_min,
  structured_data,   -- jsonb: все специфичные поля (дистанция, трассы, упражнения...)
  notes,
  feedback,          -- "easy" | "normal" | "hard"
  feedback_detail,   -- текст если "hard"
  created_at
)
```

#### 4.2 AI-тренер

**Шаг 1 — Расширенный атлетический профиль (один раз при настройке):**
```
- Виды активности и опыт в каждом
- Цели: IRONMAN 70.3, боулдеринг V6+, handstand, ...
- Слабые места и ограничения (пример: задняя миофасциальная цепь,
  левая сторона слабее, поясница, тазовое дно)
- Доступное оборудование: зал / турники / бассейн / улица
- Сколько дней в неделю готов тренироваться
- Предпочтения: что нравится, что не нравится
```

**Шаг 2 — Тесты физических показателей:**

AI предлагает пройти тесты, ты вносишь результаты:

| Тест | Что измеряет |
|---|---|
| Максимум отжиманий | Силовая выносливость (толчок) |
| Максимум подтягиваний | Силовая выносливость (тяга) |
| Планка (сек) | Стабильность кора |
| Тест Купера (12 мин бег) | Аэробная база (VO2max оценка) |
| Темп на 100м в бассейне | Плавательная база |
| Наклон вперёд (см) | Гибкость задней цепи |
| Тест Thomas | Мобильность сгибателей бедра |
| Глубокий присед | Мобильность лодыжек/бёдер |

Результаты тестов → Claude определяет уровень по каждому параметру → это становится **реальным стартовым значением** RPG-характеристик (не с нуля).

Тесты можно пересдавать раз в 4–6 недель → динамика прогресса.

**Шаг 3 — Генерация программы:**

На основе профиля + тестов + целей + расписания Claude составляет:
- Недельный сплит (какой день что тренируешь)
- Каждая тренировка: упражнения, подходы, повторения, отдых, темп
- Периодизация: недели накопления → интенсификации → разгрузки
- Прогрессия нагрузки: как и когда добавлять вес/сложность
- Если в поле **"Особенности"** атлетического профиля указаны слабые места (пример: задняя миофасциальная цепь, левая сторона слабее, поясница) — AI автоматически интегрирует коррекционные упражнения в программу. Если поле пустое — стандартная программа без акцентов.

**Программа живёт в модуле** — каждый день видишь запланированную тренировку, открываешь, делаешь по плану, отмечаешь выполнение.

**Шаг 4 — Адаптация:**

После каждой тренировки по программе — короткий фидбек:
```
Как прошло? 😰 Тяжело  😐 Нормально  😎 Легко
```
Если выбрано **😰 Тяжело** — бот задаёт уточняющий вопрос:
```
Где именно было тяжело? (можно голосом или текстом)
→ пример: "устали плечи к 3 подходу" / "не хватило дыхания на финише"
```
AI использует этот ответ для точной коррекции программы, а не просто снижает нагрузку глобально.

Если ты пропустил тренировку или добавил незапланированную активность (пошёл на джиу, покатался на велике) — AI учитывает это при планировании следующих дней.

**Шаг 5 — AI учитывает всё:**

AI-тренер видит весь контекст из других модулей:
- Плохо спал (из дневника) → снизит нагрузку на сегодня
- Не добрал белок вчера (из питания) → напомнит про восстановление
- Стресс (из настроения дневника) → предложит подвижность вместо интенсива
- Голосом сказал "завтра иду в бассейн" → AI вписывает бассейн в план и перестраивает сплит

**RPG-влияние:** тренировки качают **Выносливость** (кардио, плавание, бег, вело, джиу, скалодром) и **Силу** (зал, калистеника, турники). Соответствие программе качает **Дисциплину**.

### 05 · ПИТАНИЕ

**Функции:**
- Лог приёма пищи: завтрак / обед / ужин / перекус
- Добавление еды тремя способами:
  1. Фото этикетки → Claude читает КБЖУ и состав → автоматически вносит
  2. Фото блюда → Claude определяет блюдо и примерное КБЖУ
  3. Ручной ввод: название + КБЖУ вручную
- AI-анализ после каждого приёма: "ок / умеренно / стоит есть" + комментарий нутрициолога
- Дневной прогресс: потреблено vs норма (ККАЛ / Б / Ж / У)
- Норма берётся из физического профиля в настройках

**Виды:**
- Сегодня (все приёмы + итого)
- Дневник питания (по дням)

**Архив:**
- История питания по дням
- Графики: среднее КБЖУ за неделю/месяц, % попадания в норму

**RPG-влияние:** попадание в норму КБЖУ качает **Питание**

### 09 · ДАЙДЖЕСТ

**Функции:**
- Еженедельный AI-отчёт по всем модулям:
  - Как прошла неделя в цифрах
  - Что было хорошо, что провалено

**Периодичность:** каждое воскресенье бот присылает напоминание "оцени неделю"

**В недельном отчёте:** колесо недели + сравнение с прошлой неделей + AI-комментарий

**Быстрые команды:**
```
/tasks    — открыть задачи
/rituals  — отметить ритуалы
/food     — добавить приём пищи
/water    — добавить воду

/idea     — записать идею
/stats    — мои статы сегодня
/plan     — что запланировано на сегодня/завтра
/report   — недельный отчёт (или автоматически в вск в 20:00)
```

**Inline-кнопки под полем ввода** (Reply Keyboard, всегда видны):
```
[ ✅ Задача ] [ 🍽 Еда ] [ 💪 Тренировка ] [ 📖 Дневник ] [ 💡 Идея ] [ 📊 Статы ]
[ 😴 Сон ] [ 💰 Расход ] [ 📥 Inbox ] [ 📐 Замеры ]
```
Нажал кнопку — бот спрашивает что именно добавить. Без команд, без памяти что такое слэш.

**AI-сценарии:**

**1. Фото этикетки/блюда**
→ Claude Vision анализирует → КБЖУ + AI-комментарий нутрициолога → предлагает сохранить в лог питания

**2. Голос → действие (универсальный парсер)**

Whisper транскрибирует → Claude определяет интент → распределяет по модулям:

```
"завтра иду в бассейн и урок английского, потом работа"
→ ТРЕНИРОВКИ: плавание, завтра, запланировано
→ ВСТРЕЧИ: урок английского, завтра
→ ЗАДАЧИ: работа (или встреча, если есть время)

"сегодня пробежал 8км за 45 минут, чувствовал себя хорошо"
→ ТРЕНИРОВКИ: бег, 8км, 45мин, лог + фидбек "нормально"

"добавь задачу купить протеин до пятницы, высокий приоритет"
→ ЗАДАЧИ: купить протеин, дедлайн пятница, приоритет 🔴

"запиши в дневник: сегодня был продуктивный день, настроение 4"
→ ДНЕВНИК: текст + mood=4

"болит поясница, не хочу сегодня тяжело тренироваться"
→ AI-тренер: переключает сегодняшнюю тренировку на подвижность/растяжку
   + записывает заметку в дневник
```

**3. Умное планирование через голос**

Говоришь планы на день/неделю в свободной форме → AI разбирает и раскладывает по модулям:
- Активности → тренировки (с типом)
- События с людьми → встречи
- Дела → задачи
- Намерения/мысли → дневник или идеи

Перед сохранением бот показывает итоговый список "Я правильно понял?" → ты подтверждаешь или правишь.

**4. AI-анализ питания**
→ Claude анализирует каждый приём пищи в контексте дня + тренировки + нормы → нутрициологический комментарий

**5. Недельный отчёт (автоматически каждое воскресенье в 20:00)**

## 7. RPG-система: правила начисления XP

```
СИЛА:
  +5 XP  за каждую силовую тренировку (зал, калистеника, турники)
  +2 XP  за каждые 10 кг суммарного объёма (кг × повторения)
  +3 XP  за личный рекорд в упражнении

ВЫНОСЛИВОСТЬ:
  +5 XP  за каждую кардио-тренировку (бег, плавание, вело, джиу, скалодром)
  +3 XP  за каждые 5 км дистанции (бег/вело)
  +3 XP  за каждые 500м в бассейне
  +4 XP  за тренировку по программе (соответствие плану)

ПИТАНИЕ:
  +5 XP  за день в норме ±10% по калориям
  +3 XP  за день в норме по белку
  +2 XP  за день с нормой воды
  -3 XP  за пропущенный день без записей (не выходной)

ДИСЦИПЛИНА:
  +2 XP  за каждый выполненный ритуал
  +5 XP  за streak 7/7 ритуалов за неделю
  +3 XP  за задачу выполненную в срок
  -2 XP  за задачу просроченную более 2 дней
  +2 XP  за разобранный Inbox (0 непрочитанных)
  +3 XP  за регулярный сон 7–9ч (5+ дней в неделю)

РЕФЛЕКСИЯ:
  +4 XP  за запись в дневник
  +5 XP  за запись с настроением
  +10 XP за достигнутую цель
  +3 XP  за заполненное колесо баланса за неделю
  +2 XP  за замеры тела (раз в 2 недели)
  +1 XP  за запись расхода в финансы (макс 3 в день)

УРОВЕНЬ = floor(среднее_XP_всех_5_характеристик / 100)
Максимум XP одной характеристики: 5000
```

### Structured Outputs — гарантированный JSON

Оба провайдера поддерживают structured outputs. Без этого — постоянные try/except и баги парсинга.

```python
# Groq — парсинг голоса
from groq import Groq
from pydantic import BaseModel

class VoiceIntent(BaseModel):
    module: str   # "workout"|"food"|"task"|"diary"|"sleep"|"finance"
    action: str   # "add"|"update"|"query"
    data: dict
    confidence: float

client = Groq()
response = client.chat.completions.create(
    model="llama-3.3-70b-versatile",
    messages=[{"role": "user", "content": transcribed_text}],
    response_format={"type": "json_object"}
)

# Gemini — фото еды
import google.generativeai as genai
model = genai.GenerativeModel("gemini-2.5-flash")
```

## 12. Supabase RLS — безопасность с первого дня

Без RLS при любом баге в коде данные одного пользователя могут утечь другому. С RLS — это невозможно на уровне БД. Настраивается один раз при создании схемы.

```sql
-- Применяется к КАЖДОЙ таблице с user_id
ALTER TABLE tasks ENABLE ROW LEVEL SECURITY;
CREATE POLICY "user_owns_tasks" ON tasks
  FOR ALL USING (user_id = auth.uid())
  WITH CHECK (user_id = auth.uid());

-- Исключение: таблица exercises (общая база упражнений, публичная)
ALTER TABLE exercises ENABLE ROW LEVEL SECURITY;
CREATE POLICY "exercises_public_read" ON exercises
  FOR SELECT USING (true);
```

**Важно для FastAPI:** бэкенд использует `service_role` ключ (обходит RLS) только для внутренних операций (отчёты, APScheduler). Запросы от пользователя — через `anon` ключ с JWT токеном.

### Как это меняет MY-OS

**Сон** — трекается автоматически, не надо вводить вручную. Фазы сна из часов → детальная статистика.

**Тренировки** — после тренировки пульс, каллории, маршрут подтягиваются автоматически. Ты только подтверждаешь запись.

**AI-тренер** — видит HRV утром → определяет уровень восстановления → корректирует нагрузку:
```

HRV ниже на 25%+          → "День восстановления, никаких нагрузок"
```

**RPG** — характеристики растут на основе реальных биометрических данных, не только ручного ввода.

**Дашборд** — живые данные с часов: пульс прямо сейчас, шаги за день, уровень стресса.

## 15. Онбординг

Поток первого входа в MY-OS. Проходится один раз, пропустить нельзя (кроме шага 3).

---

### Шаг 1 — Приветствие

```
Экран:

        MY-OS ©
   my operating system

  Твоя личная ОС для жизни.
  Трекай всё что важно —
  тренировки, питание, задачи,
  привычки, финансы и рост.

        [ Начать ]
```

---

### Шаг 2 — Физический профиль

Поля (все обязательные):
- Имя (как обращаться)
- Пол
- Дата рождения (возраст считается автоматически)
- Вес (кг)
- Рост (см)
- Цель: похудеть / набрать массу / поддерживать / спортивная форма

На основе этого Claude автоматически рассчитывает:
- Суточная норма калорий (формула Миффлина-Сан Жеора + коэффициент активности)
- Норма БЖУ под цель
- Норма воды

Показываем результат: "Твоя норма: **2743 ккал · Б168 / Ж76 / У345**" → можно скорректировать вручную.

---

### Шаг 3 — Атлетический профиль *(можно пропустить)*

```
        [ Пропустить → ]
```

Если заполняет:
- Виды активности (мультиселект): бег / плавание / зал / боулдеринг / джиу / муай-тай / велосипед / другое
- Опыт: новичок / любитель / продвинутый
- Главная цель: выносливость / сила / гибкость / снижение веса / конкретное событие (IRONMAN, соревнование...)
- Особенности и ограничения: свободный текст ("левая сторона слабее, поясница")
- Дней в неделю для тренировок: 1–7

AI-тренер становится доступен сразу после заполнения.

Выбранные ритуалы сразу появляются в модуле 02 с дефолтными настройками. Время напоминаний можно поменять потом.

```
        MY-OS готов.

  ◆ Профиль заполнен
```

### Технические детали онбординга

- Хранится флаг `users.onboarding_completed: bool`
- Если `false` → при открытии Mini App редирект на онбординг
- Шаги 1–2–4 обязательные, шаг 3 пропускаемый
- Прогресс онбординга сохраняется — если закрыл на шаге 2, при следующем открытии продолжит с шага 2
- После онбординга в БД: заполнен `users`, создан `user_stats` (все 0), созданы `rituals` из выбранных
